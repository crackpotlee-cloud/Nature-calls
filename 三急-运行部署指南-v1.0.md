## 第一部分：快速体验（5分钟上手）

想在部署前快速体验项目效果？按以下步骤操作，无需安装任何依赖。

### 1.1 查看交互原型

直接用浏览器打开高保真交互原型：

```bash
# macOS
open /workspace/三急-交互原型/index.html

# Linux
xdg-open /workspace/三急-交互原型/index.html

# Windows
start /workspace/三急-交互原型/index.html
```

或直接拖拽 `三急-交互原型/index.html` 文件到浏览器窗口。

### 1.2 打开审核后台

审核后台是一个单文件 SPA，无需服务器即可运行：

```bash
# macOS
open /workspace/admin/index.html

# Linux
xdg-open /workspace/admin/index.html

# Windows
start /workspace/admin/index.html
```

> **Mock 管理员账号**：
> - `admin_l3` — 大区管理员（可审核、锁定、编辑）
> - `admin_l4` — 平台运营（全部权限）
>
> ⚠️ 以上仅用于开发测试。生产环境必须接入真实后端认证。

### 1.3 运行后端引擎测试

```bash
cd /workspace/backend

# 测试状态计算引擎（6个案例）
python3 status_engine.py

# 测试推荐算法引擎（4个场景）
python3 recommend_engine.py
```

**预期输出**：
- `status_engine.py`：6个测试案例，覆盖 GREEN/YELLOW/RED 全部状态
- `recommend_engine.py`：4个推荐场景（智能急、智能急夜间、腹泻急、带娃急）

> 如果案例1（IFS 5min反馈）返回 YELLOW 而非 GREEN，说明当前服务器时区不在商场营业时间内（UTC时区差异），代码逻辑本身正确。

### 2.3 数据库初始化

#### 创建数据库

```bash
# 连接 PostgreSQL
sudo -u postgres psql

# 在 psql 中执行
CREATE USER sanji WITH PASSWORD 'your_secure_password';
CREATE DATABASE sanji OWNER sanji;
\c sanji
CREATE EXTENSION IF NOT EXISTS postgis;
CREATE EXTENSION IF NOT EXISTS "uuid-ossp";
\q
```

#### 执行建表脚本

```bash
PGPASSWORD=your_secure_password psql -U sanji -d sanji -f /workspace/backend/schema.sql
```

建表脚本会创建以下表：
- `toilets` — 厕所主表（含 PostGIS 空间索引）
- `feedbacks` — 用户反馈表
- `users` — 用户表
- `corrections` — 纠错表

同时会创建触发器（自动更新 `updated_at`、自动填充几何列）和 12 个索引。

#### 导入 Mock 数据

```bash
PGPASSWORD=your_secure_password psql -U sanji -d sanji -f /workspace/backend/mock_data_insert.sql
```

Mock 数据包含：

- **2011 条反馈**
- **80 个用户**（L1~L4 角色分布）

### 2.4 状态计算引擎使用方法

`status_engine.py` 是三色状态灯的计算核心。它接收厕所信息和反馈列表，输出状态颜色与置信度。

#### 命令行测试

```bash
python3 /workspace/backend/status_engine.py
```

#### Python 代码调用

```python
from datetime import datetime
from status_engine import (
    ToiletInfo, FeedbackRecord, calculate_status, batch_calculate
)

# 定义厕所信息

    toilet_id="toilet-001",
    name="IFS 3F 海底捞旁",
    type="商场厕所",
    operating_type="跟随商场",
    open_time=time(10, 0),
    close_time=time(22, 0),
    is_mall=True,
)

# 定义反馈列表（按时间倒序）

    FeedbackRecord(
        status="available",

        trust_score=85

]

# 计算状态
result = calculate_status(toilet, feedbacks)
print(f"状态: {result.color.value}")   # green/yellow/red
print(f"置信度: {result.confidence}")   # 0-100
print(f"原因: {result.reason}")

# 批量计算
```

**状态计算规则速查**：

| 条件 | 状态 | 置信度 |
|------|------|--------|
| 5分钟内有可用反馈 | 🟢 GREEN | 95 |
| 1小时内有可用反馈（非5分钟内） | 🟢 GREEN | 60-90（衰减） |

### 2.6 API 服务启动（使用 Flask）

引擎代码是可独立调用的 Python 模块，需要包装为 HTTP API 供小程序调用。

#### 最小化 Flask API 示例

创建文件 `/workspace/backend/app.py`：

```python
#!/usr/bin/env python3
"""
三急 API 服务 - 最小化启动示例
"""
from flask import Flask, request, jsonify
from status_engine import ToiletInfo, FeedbackRecord, calculate_status
from recommend_engine import UserContext, ToiletCandidate, SceneMode, recommend
import psycopg2
import os
from datetime import datetime

app = Flask(__name__)

# 数据库连接
DB_URL = os.environ.get("DATABASE_URL", "postgresql://sanji:password@localhost:5432/sanji")

def get_db():
    return psycopg2.connect(DB_URL)

# ===== 附近厕所搜索 =====
@app.route("/v1/toilets/nearby")
def toilets_nearby():
    lat = float(request.args.get("lat", 0))
    lng = float(request.args.get("lng", 0))
    radius = int(request.args.get("radius", 2000))
    page = int(request.args.get("page", 1))
    page_size = int(request.args.get("page_size", 20))

    conn = get_db()
    cur = conn.cursor()
    cur.execute("""
        SELECT id, name, type, lat, lng, entry_lat, entry_lng,
               landmark, floor, direction, access_type,
               has_baby_station, has_accessible, has_paper,
               has_soap, has_hand_dryer,
               operating_type, open_time, close_time,
               status_color, status_confidence,
               ST_Distance(entry_geom, ST_SetSRID(ST_MakePoint(%s, %s), 4326)::geography) AS distance_m
        FROM toilets
        WHERE ST_DWithin(entry_geom, ST_SetSRID(ST_MakePoint(%s, %s), 4326)::geography, %s)
        ORDER BY distance_m
        LIMIT %s OFFSET %s
    """, (lng, lat, lng, lat, radius, page_size, (page - 1) * page_size))

    items = []
    for row in cur.fetchall():
        items.append({
            "id": str(row[0]),
            "name": row[1],
            "type": row[2],
            "lat": float(row[3]), "lng": float(row[4]),
            "entry_lat": float(row[5]), "entry_lng": float(row[6]),
            "landmark": row[7], "floor": row[8],
            "direction": row[9], "access_type": row[10],
            "has_baby_station": row[11], "has_accessible": row[12],
            "has_paper": row[13], "has_soap": row[14],
            "has_hand_dryer": row[15],
            "operating_type": row[16],
            "open_time": str(row[17]) if row[17] else None,
            "close_time": str(row[18]) if row[18] else None,
            "status_color": row[19],
            "status_confidence": row[20],
            "distance_m": round(float(row[21])),
            "walk_time_min": round(float(row[21]) / 80.0, 1),
        })

    return jsonify({"code": 0, "data": {"total": len(items), "page": page, "page_size": page_size, "items": items}})

# ===== 智能推荐 =====
@app.route("/v1/recommend", methods=["POST"])

    lat = data.get("lat", 0)
    lng = data.get("lng", 0)
    scene = data.get("scene", "smart")
    radius = data.get("radius", 2000)
    top_k = data.get("top_k", 5)

    # 从数据库查询候选厕所并转换为 ToiletCandidate
    # ... (实现省略，参照 status_engine 和 recommend_engine 调用)

    return jsonify({"code": 0, "data": {"scene": scene, "recommendations": []}})

# ===== 提交反馈 =====
@app.route("/v1/feedbacks", methods=["POST"])
def submit_feedback():
    data = request.json
    # 存储反馈 + 异步触发状态更新
    return jsonify({"code": 0, "data": {"feedback_id": "uuid", "trust_score_earned": 2}})

if __name__ == "__main__":
    app.run(host="0.0.0.0", port=8000, debug=True)
```

### 4.2 方式 B：部署到静态服务器

#### Nginx 部署

```bash
# 复制文件到 Nginx 目录
sudo cp /workspace/admin/index.html /var/www/html/sanji-admin/

# Nginx 配置示例 (/etc/nginx/conf.d/sanji-admin.conf)
server {
    listen 80;
    server_name admin.sanji.app;

    root /var/www/html/sanji-admin;
    index index.html;

    location / {
        try_files $uri $uri/ /index.html;
    }

    # 如果审核后台调用了后端 API
    location /api/ {
        proxy_pass http://localhost:8000/;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }
}

# 重载 Nginx
sudo nginx -t && sudo nginx -s reload
```

## 第五部分：切换到生产模式

### 5.1 Mock 数据 → 真实数据库切换

| 步骤 | 操作 | 文件 |
|------|------|------|

| 2 | 将 `USE_MOCK` 改为 `false` | `miniprogram/utils/api.js` |
| 3 | 将 `BASE_URL` 改为生产地址 | `miniprogram/utils/api.js` |

### 5.2 腾讯地图 Key 申请

1. 访问 [腾讯位置服务](https://lbs.qq.com/)
2. 注册/登录账号
3. 进入「控制台」→「应用管理」→「创建应用」
4. 应用类型选择「微信小程序」
5. 填写小程序 AppID
6. 勾选以下服务：
   - WebService API（地址解析、逆地址解析）
   - 微信小程序 SDK
7. 获取 Key 后填入 `pages/index/index.js` 的 `mapKey` 字段
8. 在小程序后台添加 request 合法域名：`https://apis.map.qq.com`

### 5.3 微信小程序 AppID 注册

1. 访问 [微信公众平台](https://mp.weixin.qq.com/)
2. 点击「立即注册」→ 选择「小程序」
3. 按要求填写邮箱、密码等信息
4. 选择主体类型（企业/个体工商户/个人）
5. 完成微信认证（企业主体需支付 300 元/年认证费）
6. 进入「开发」→「开发管理」→「开发设置」
7. 获取 AppID 和 AppSecret
8. 配置服务器域名白名单

### 5.4 小程序审核注意事项

#### 名称"三急"的审核说明

小程序名称"三急"可能触发审核关注，建议准备以下材料：

1. **名称解释**：在提交审核时附上名称说明——"三急"取自中国传统文化中的"人有三急"，产品定位为厕所实时资源调度引擎，帮助用户快速找到可用厕所。
2. **备用名称**（如审核不通过）：
   - 急厕导航
   - 急寻厕所
   - 厕所在哪
   - 厕所导航
3. **类目选择**：建议选择「工具」→「信息查询」或「生活服务」→「便民服务」

#### 内容审核要点

- 所有厕所相关图片需避免不雅内容
- 分享卡片文案需符合微信分享规范
- 确保隐私政策完整（不收集个人信息，仅用 OpenID 匿名）

## 第六部分：常见问题

### 数据库相关

**Q: psql 命令提示 "command not found"？**

A: PostgreSQL 客户端未安装。安装方法：

```bash
# macOS
brew install libpq && echo 'export PATH="/opt/homebrew/opt/libpq/bin:$PATH"' >> ~/.zshrc

# Ubuntu
sudo apt install postgresql-client-15
```

**Q: 导入 Mock 数据时报错 "relation 'toilets' does not exist"？**

A: 建表脚本未执行。请先执行 `schema.sql`：

```bash
PGPASSWORD=your_password psql -U sanji -d sanji -f /workspace/backend/schema.sql
```

**Q: PostGIS 扩展创建失败？**

A: 确保 PostgreSQL 已安装 PostGIS 扩展：

```bash
# Ubuntu
sudo apt install postgresql-15-postgis-3

# 然后在数据库中执行
psql -U sanji -d sanji -c "CREATE EXTENSION IF NOT EXISTS postgis;"
```

### 后端引擎相关

**Q: 运行 `status_engine.py` 时报错 "No module named 'dataclasses'"？**

A: `dataclasses` 是 Python 3.7+ 标准库。检查 Python 版本：

```bash
python3 --version
# 需要 ≥ 3.7，推荐 ≥ 3.11
```

**Q: 状态引擎案例1返回 YELLOW 而非 GREEN？**

A: 这是时区问题。沙箱/服务器在 UTC 时区下，当前时间对应的北京时间为商场闭店时段（22:00后）。部署到生产环境（Asia/Shanghai 时区）后正常。确认服务器时区：

```bash
timedatectl
# 确保 Time zone: Asia/Shanghai (CST, +0800)
```

**Q: 如何自定义推荐算法的权重？**

A: 编辑 `recommend_engine.py` 中的 `recommend()` 函数，修改各场景下的权重字典：

```python
# 智能急权重（第 354-359 行）
weights = {
    'arrival': 0.40,
    'availability': 0.30,
    'quality': 0.20,
    'scene_match': 0.10,
}
```

### 小程序相关

**Q: 微信开发者工具编译报错 "未找到 app.json 入口"？**

A: 确认项目目录指向正确路径：

```bash
# 正确路径
/workspace/miniprogram/

# 而非
/workspace/
```

**Q: 地图不显示或显示空白？**

A: 可能原因：
1. 未配置腾讯地图 Key → 参考步骤 4
2. 未在小程序后台添加地图 API 域名白名单 → 添加 `https://apis.map.qq.com`
3. 微信开发者工具版本过低 → 更新到最新版

**Q: wx.openLocation 调用无反应？**

A: 
1. 确认微信版本 ≥ 8.0
2. 开发者工具中点击「模拟器」右上角「…」→ 开启「位置模拟」
3. 真机上检查 GPS 是否已开启

**Q: 如何区分开发环境和生产环境的 API 地址？**

A: 在 `utils/api.js` 中：

```javascript
// 开发环境：使用 Mock 数据，无需后端
const USE_MOCK = true
const BASE_URL = ENV.development

// 生产环境：使用真实后端
const USE_MOCK = false
const BASE_URL = ENV.production
```

### 审核后台相关

**Q: 审核后台登录提示"该ID无管理员权限"？**

A: 开发/Mock 模式下仅 `admin_l3` 和 `admin_l4` 可登录。生产环境需接入真实后端认证。
