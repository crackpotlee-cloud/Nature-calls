# 三急 · 本地电脑验证指南 v1.0

> 下载项目文件后，在你的本地电脑上直接运行验证。

---

## 第一步：下载项目文件

从项目资料库下载 `三急-完整项目源码.tar.gz`，解压到任意目录：

```bash
tar -xzf 三急-完整项目源码.tar.gz
cd 三急-完整项目源码  # 或解压后进入对应目录
```

解压后的目录结构：

```
├── miniprogram/          # 微信小程序前端（需微信开发者工具）
├── backend/              # 后端Python引擎
├── admin/                # 审核后台（浏览器直接打开）
├── 三急-交互原型/         # 交互原型（浏览器直接打开）
└── *.md                  # 全部文档
```

---

## 第四步：验证后端引擎（需要Python 3）

### 4.1 确认Python环境

```bash
python3 --version
# 需要 Python 3.9+
```

### 4.2 运行状态引擎测试

```bash
cd backend
python3 status_engine.py
```

**你会看到6个测试用例的输出：**
```
[案例1] 5min内有可用反馈 → GREEN 95 ✓
[案例2] 1h内≥3人报告不可用 → RED 85 ✓
[案例3] >3h无反馈 → YELLOW 40 ✓
[案例4] 商场闭店 → YELLOW 50 ✓
[案例5] 地铁末班车后 → YELLOW + "运营结束" ✓
[案例6] 30分钟前反馈衰减 → GREEN 79 ✓
```

### 4.3 运行推荐引擎测试

```bash
cd backend
python3 recommend_engine.py
```

**你会看到3个场景的推荐结果：**
```
[场景1] 智能急模式 → IFS 3F 海底捞旁 得分 0.94
[场景2] 腹泻急模式（硬筛选有纸）→ 4个候选
[场景3] 带娃急模式（硬筛选母婴台）→ 2个候选
```

### 4.4 启动后端API服务

```bash
cd backend
pip3 install flask          # 安装Flask（一次性）
python3 server.py
```

启动后：
```
🚀 三急后端API: http://0.0.0.0:5000 (6个厕所)
```

新开一个终端测试：

```bash
# 测试健康检查
curl http://localhost:5000/health

# 测试推荐接口
curl -X POST http://localhost:5000/api/toilet/nearby \
  -H "Content-Type: application/json" \
  -d '{"lat":30.6586,"lng":104.0817,"scene":"smart"}'
```

## 第五步：验证微信小程序（需要微信开发者工具）

### 5.1 下载微信开发者工具

官网下载：https://developers.weixin.qq.com/miniprogram/dev/devtools/download.html

安装后打开。

### 5.2 导入项目

1. 微信开发者工具 → 选择「导入项目」
2. 项目目录：选择 `miniprogram/` 文件夹
3. AppID：使用你自己的测试AppID（或选择「测试号」）
4. 点击「导入」

### 5.3 预览运行

导入后微信开发者工具会自动编译，你可以在模拟器中看到：

- **首页**：全屏地图 + 底部"急"按钮
- 点击"急" → 推荐结果
- 点击"开始导航" → 导航页
- 点击"我到了" → 反馈页

**注意**：小程序使用Mock数据模式，不需要后端服务。在 `utils/api.js` 中 `USE_MOCK = true`。

---

## 第六步：切换到真实后端

当你想让小程序调用后端API而非Mock数据时：

1. 确保后端 `python3 server.py` 已启动在 `localhost:5000`
2. 修改 `miniprogram/utils/api.js`：
   ```javascript
   const USE_MOCK = false;          // 关闭Mock
   const BASE_URL = 'http://localhost:5000';  // 后端地址
   ```
3. 微信开发者工具中重新编译

## 常见问题

**Q: 交互原型页面空白？**
A: 确认文件路径正确，浏览器控制台查看是否有JS错误。

**Q: Python报错 `ModuleNotFoundError: No module named 'flask'`？**
A: 运行 `pip3 install flask` 安装Flask。

**Q: 微信开发者工具导入后编译失败？**
A: 检查 `project.config.json` 中的 `appid` 是否有效，或切换为「测试号」。

**Q: 后端API启动后curl连不上？**
A: 确认 `python3 server.py` 正常运行中，端口5000未被占用。

**Q: 小程序页面没有数据？**
A: 确认 `utils/api.js` 中 `USE_MOCK = true`，Mock数据内嵌在代码中。
