# 三急 v1.2 修复说明

## 已修复的问题

### 1. 移除所有 hardcoded 逻辑（超出POI坐标的部分）
**文件**: `miniprogram/utils/api.js`

- 删除了 `NEARBY_EXTRA` 硬编码数组（写死的距离和步行时间）
- `getNearbyToilets()`: 现在根据用户真实坐标动态计算 `distance_m` 和 `walk_time_min`
- `recommendToilet()`: 移除写死的 `score` 数组 `[0.925, 0.78, ...]`，改为根据距离/状态/设施动态计算
- `detailToRecommendation()`: `score_breakdown` 改为动态计算，不再写死
- `getToiletsByDistrict()`: 同样使用动态距离计算

### 2. 地图中心改为用户GPS定位
**文件**: `miniprogram/app.js`, `miniprogram/pages/index/index.js`, `miniprogram/pages/result/result.js`, `miniprogram/pages/nav/nav.js`

- `app.js` `getLocation()`: 失败时在控制台输出警告（不再静默使用成都）
- `index.js`: 新增 `initLocation()` 和 `useLocation()` 方法，等待真实GPS后再加载地图
- `result.js` / `nav.js`: 默认坐标改为 0（等待真实位置），`onLoad` 中使用 `globalData.currentLocation`
- 所有页面不再硬编码 `latitude: 30.658, longitude: 104.082` 作为初始值

### 3. 路线规划使用真实地图API（传递起终点+步行模式）
**文件**: `miniprogram/utils/map.js`

- `fetchWalkingRouteFromTencent()` 正确实现了腾讯地图 polyline 压缩格式解压
- 解压逻辑: `rawPolyline[0]/1000000 = 起点lat`, `rawPolyline[1]/1000000 = 起点lng`, 后续每两个元素为相对偏移量
- 增加了 `console.log` 方便调试 API 返回

### 4. 路线不再显示为直线（根本原因分析）
**根本原因**: API Key `5CRBZ-IFJW7-YE7XS-HBX5B-R2HSJ-SLBCC` **未开启 WebServiceAPI 功能**

用 curl 测试结果:
```json
{
  "status": 199,
  "message": "此key未开启WebserviceAPI功能"
}
```

API 调用返回 199 错误 → 代码走到 `catch` → 降级为直线连接两点

## 需要您完成的配置（必须！）

### 步骤1: 开启 WebServiceAPI 权限
1. 访问 https://lbs.qq.com/
2. 登录，进入「控制台」
3. 找到 Key: `5CRBZ-IFJW7-YE7XS-HBX5B-R2HSJ-SLBCC`
4. 点击「修改」，勾选启用「WebServiceAPI」产品
5. 保存

### 步骤2: 配置小程序合法域名
1. 访问 https://mp.weixin.qq.com/
2. 登录小程序管理员账号
3. 进入「开发」→「开发管理」→「开发设置」→「服务器域名」
4. 在 `request 合法域名` 中添加: `https://apis.map.qq.com`
5. 点击「保存」

### 步骤3: 开发工具临时绕过（仅开发阶段）
如果不想立即配置合法域名，可以在微信开发者工具中:
1. 点击右上角「详情」
2. 找到「本地设置」→ 勾选「不校验合法域名、web-view、TLS版本以及HTTPS」
3. 这样 `wx.request` 可以直接访问 `apis.map.qq.com`

## 验证修复是否生效

配置完成后，在小程序中:
1. 点击首页「急」按钮
2. 进入推荐结果页，查看地图中的路线
3. 如果路线显示为**沿道路的曲线**（不是直线），则修复成功
4. 打开调试控制台，应该看到: `[map.js] 腾讯direction API 响应: ...` 和 `[map.js] 解压成功, 共XX个点`
